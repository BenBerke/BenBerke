## Main Project

### [Tilky Engine](https://github.com/BenBerke/Tilky-Engine)

Tilky Engine is my open-source **sector-based 3D game engine and editor** built in C++20.

It includes a full toolchain:

* Project launcher
* Visual level editor
* OpenGL runtime
* Lua scripting
* OpenAL positional audio
* Data-oriented ECS
* Sector-aware physics and collision
* Tracy profiling
* Rust-based exporting and localisation tooling
* Standalone export support

The renderer is inspired by classic Doom/Build-style engines, using connected sectors, independent floor and ceiling heights, textured walls and floors, decals, billboards, sprites, depth buffering, and SSBO-backed geometry buffers.

## Other Projects

### [3D Software Renderer](https://github.com/BenBerke/3D-Software-Renderer)

A CPU-side textured 3D renderer written in C.

Features include:

* Custom math structures
* World-to-camera transforms
* Clipping
* Perspective projection
* Depth buffering
* Textured triangle and quad rendering
* UV interpolation and texture sampling

### [C Interpreter](https://github.com/BenBerke/C-Interpreter)

A tree-walking interpreter written in C.

Includes:

* Hand-written lexer
* Recursive-descent parser
* AST representation
* Variables and expressions
* Conditionals
* Scoped blocks
* Return values
* User-defined functions
* Recursive function calls
* Manual AST/runtime memory management

### Custom CPU / VM Work

I also work on CPU architecture, virtual machines, and low-level execution models, including digital logic CPU experiments and Rust-based virtual machine projects.
